## Decision

Replaced the old substring-search approach in read_knowledge with a target-based index lookup. The previous implementation scanned entry bodies for substring matches, which was redundant with the structured data already available in modules.json and files.json. The new approach resolves the target against modules.json first (exact module match), then falls back to files.json (file path match). This is faster, more predictable, and aligns with the tool's documented behavior in the spec.

## Alternatives

- Substring search across entry bodies — rejected because it duplicated what the _graph/ indexes already provide and produced unpredictable matches

## Assumptions

- _graph/ indexes are always fresh via rebuildAll after every write_knowledge call
