## Decision

Introduced a rebuildAll() function that orchestrates both buildIndexes() and generateIndexMd() in sequence. Previously, generateIndexMd was defined but never called from anywhere, meaning index.md would go stale. Now every write path (mcp/server's write_knowledge and cli/reindex) calls rebuildAll, ensuring both _graph/ JSON indexes and index.md stay in sync after every mutation.

## Risk

If rebuildAll is bypassed (e.g. manual file edits to .knowledge/), both _graph/ indexes and index.md will drift out of sync. Always use rebuildAll or run kb reindex after manual changes.
